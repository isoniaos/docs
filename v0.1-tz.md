# IsoniaOS v0.1 — Техническое задание

## 1. Цель

Создать **private prototype** governance architecture layer для EVM + TypeScript stack.

v0.1 должен доказать, что:

- shared protocol deployment может обслуживать множество организаций;
- governance-структура формализуется через `Organization`, `Body`, `Role`, `Mandate`, `PolicyRule`, `Proposal`;
- control plane надёжно индексирует chain events;
- frontend показывает карту власти и route принятия решения;
- архитектура готова к self-hosted и SaaS режимам.

## 2. Ключевые решения

- one deployment → many organizations
- `orgId` as hard isolation boundary
- REST as public API for v0.1
- event-first ingestion
- public app-core
- private SaaS overlay
- replaceable default theme
- minimal npm dependencies

## 3. Входит в v0.1

### On-chain

- `GovCore`
- `GovProposals`
- `DemoTarget`
- organizations
- bodies
- roles
- mandates
- policy rules
- proposal lifecycle

### Off-chain

- blockchain ingestion
- raw event storage
- projections
- REST API
- route explanation
- governance graph
- idempotent processing
- replay support

### Frontend

- org list
- governance overview
- proposals list/details
- route explanation UI
- governance map
- theme separation

## 4. Не входит в v0.1

- token voting
- anti-Sybil
- reputation
- marketplace
- enterprise SSO
- billing core
- AI copilot
- arbitrary execution engine
- proxy upgradeability

## 5. Acceptance criteria

### Protocol

- можно создать минимум 2 организации
- у организаций изолированные bodies/roles/mandates/proposals
- есть минимум 3 proposal types
- proposal lifecycle работает end-to-end
- execution меняет state demo target

### Control Plane

- события ingest'ятся идемпотентно
- projections можно rebuild
- API отдаёт route explanation и governance graph
- reprocessing event не ломает состояние
