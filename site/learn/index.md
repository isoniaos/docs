# Learn IsoniaOS

This section explains the IsoniaOS product model in plain English.

Use it when you want to understand what IsoniaOS does before reading technical details.

## Core Idea

IsoniaOS treats governance as a lifecycle:

```text
Proposal -> Review -> Decision -> Execution -> Evidence -> Accountability -> Memory
```

A decision is not finished when it passes. The organization still needs to know who owns the next step, what happened, what proof exists, and what future participants should remember.

## Example Used In This Guide

The pages use a fictional example called Community Grants DAO.

Community Grants DAO wants to fund contributors in a clearer way. It creates an organization, chooses a simple grants template, activates the required settings, approves a contributor proposal, records execution evidence, and tracks whether the work was completed.

The example is illustrative. It does not claim that every step is complete in the current developer-preview build.
