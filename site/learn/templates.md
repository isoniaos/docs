# Templates

Templates are reusable governance patterns.

They help an organization start with a known structure instead of designing every process from zero.

## What A Template Can Define

A template can include:

- roles;
- permissions;
- required proposal fields;
- review steps;
- approval route;
- execution expectations;
- evidence requirements;
- accountability fields;
- common statuses.

## Why Templates Matter

Most organizations repeat similar patterns:

- fund contributor work;
- approve a working-group mandate;
- change a technical setting;
- review a grant milestone;
- confirm a completed action.

Templates help the organization make those patterns clear and repeatable.

## Choosing A Template

Choose the simplest template that matches the decision type.

| If the organization needs to... | Start with... |
| --- | --- |
| Fund contributor work | A grants or contributor funding template. |
| Create a working group | A mandate template. |
| Approve a technical change | A technical change template. |
| Track a milestone | A milestone review template. |
| Record a simple decision | A basic proposal template. |

## Example

Community Grants DAO chooses a grants template. The template asks for the proposed scope, requested budget, responsible contributor, review group, expected evidence, due date, and completion review.

The template does not decide for the organization. It makes the decision path easier to follow.
