# Glossary

| Term | Meaning |
| --- | --- |
| Accountability record | A record connecting a decision to responsible parties, status, due dates, evidence, and completion or failure notes. |
| App Core | The self-hostable IsoniaOS React/Vite governance console. |
| Authority claim | A statement about whether a source is authoritative for a specific field or only evidence/context. |
| Control Plane | The indexing, projection, diagnostics, and REST API service. |
| Evidence | A transaction, provider record, external link, manual update, or other artifact that supports a governance claim. |
| Governance lifecycle | The flow from proposal and review through decision, execution, accountability, and memory. |
| Integration Lab | Isolated repository for provider validation, Sepolia workflows, evidence templates, and scenarios. |
| Proposal route | The approval, veto, timelock, and executor path that applies to a proposal type. |
| Read model | A projected API/database view derived from raw events or other sources. |
| Source disclosure | Metadata showing source label, trust boundary, authority claim, verification method, and data status. |
| Trust boundary | The point where a source's claims stop being authoritative and become evidence, context, or annotation. |
