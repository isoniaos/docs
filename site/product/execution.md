# Execution

Execution is where governance decisions become actions.

IsoniaOS should help organizations connect approved intent to what actually happened.

## Execution data

Execution views may include:

- expected target or action;
- responsible party;
- route or approval source;
- due date;
- transaction hash or proof link;
- execution status;
- failure or cancellation reason;
- mismatch warnings;
- completion review.

## Contract-backed execution

When execution is modeled in contracts, contract state and events are authoritative for the fields they define.

## External execution

Many organizations execute through existing tools such as multisigs, Governor systems, offchain operations, contributor work, grant milestones, or public reports.

Those records should be treated as evidence or context unless a specific integration verifies and models a field as authoritative.

## Manual execution updates

Manual updates can document progress, blockers, or completion notes. They must stay labeled as annotations.
