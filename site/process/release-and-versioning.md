# Release and Versioning

Active public docs should stay current and readable.

## Active docs

The active tree does not keep version clutter. Old docs should not remain in active archive directories only for history.

## History

Old documentation remains available through Git history and release tags.

## Tags

Release tags are created after coherent validated waves. They should not be created as normal churn for every small cross-repository update.

## Public docs changes

Docs changes should update `CHANGELOG.md` under `Unreleased` until a release is intentionally prepared.

## Claim discipline

Version labels do not imply production readiness, audit readiness, provider-completeness, public beta readiness, legal readiness, or launch readiness. Those claims require scoped evidence gates.
