# Execution And Evidence

Execution is the action expected after a decision.

Evidence is the record that supports what happened.

## Execution Examples

Execution might be:

- sending a configured transaction;
- publishing an external proof;
- completing a milestone;
- starting a working-group mandate;
- recording that no action is required.

## Evidence Examples

Evidence might be:

- transaction hash;
- external public link;
- document or deliverable link;
- reviewer confirmation;
- manual completion note;
- failure or cancellation reason.

## Basic Flow

1. Open the approved decision.
2. Review the expected action.
3. Confirm who is responsible.
4. Perform or confirm the action.
5. Attach evidence.
6. Check whether the evidence supports the claim.
7. Update the accountability record.

## How To Read Evidence

Ask:

- What claim does this evidence support?
- Where did it come from?
- Is it current, stale, failed, missing, or unknown?
- Does it prove execution, or only provide context?
- Does a reviewer still need to confirm completion?

## Example

Community Grants DAO approved a contributor grant. The executor attaches a transaction hash after the approved funding action. The contributor later attaches the deliverable. A reviewer confirms whether the deliverable matches the approved scope.
