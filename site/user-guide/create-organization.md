# Create An Organization

Creating an organization starts a governance space for a DAO, working group, grants program, council, or community.

## Before You Start

Decide:

- organization name;
- short purpose;
- initial admin or admin group;
- first governance template;
- who will review proposals;
- who can approve decisions;
- who can execute approved actions;
- what evidence should prove completion.

## Basic Flow

1. Open the organization creation flow.
2. Enter the organization name and purpose.
3. Choose the initial admin.
4. Choose a template or start with a basic setup.
5. Review the required roles and settings.
6. Save the organization draft.

In current developer-preview work, creating an organization may require configured contracts, Control Plane, App Core, and a wallet connected to the expected network. If a user-facing flow is not exposed in the current build, treat this page as the product model and use the [developer overview](../developers/index.md) to find the owning repository.

## Example

Community Grants DAO creates a new organization with this purpose:

```text
Fund contributor work and preserve clear evidence of grant decisions, execution, and completion.
```

The admin chooses a grants template because the organization expects repeat funding proposals with due dates and completion review.

## Good Organization Setup

A good organization setup answers:

- What is this group for?
- Who can configure it?
- Which template is active?
- Who can review proposals?
- Who can approve decisions?
- Who can execute approved actions?
- What evidence is required?

If those answers are unclear, finish configuration before activation.
