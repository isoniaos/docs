# Activate An Organization

Activation means an organization has completed the required setup for its selected governance flow.

It is the difference between "we started configuring this" and "this organization can use the configured process."

## What Activation Checks

Activation may require:

- organization name and purpose;
- admin role;
- selected template;
- proposal route;
- review and approval roles;
- execution role or action path;
- evidence requirements;
- contract-backed state where the flow requires it.

The exact checks depend on the template and implementation. Current developer-preview work may expose only part of the intended activation flow.

## Basic Flow

1. Review the organization setup.
2. Confirm required roles.
3. Confirm proposal route.
4. Confirm execution and evidence requirements.
5. Fix missing or invalid settings.
6. Activate the organization.
7. Review the active status and any warnings.

## Why Activation Matters

Without activation, participants may think a process is usable before it is configured. Activation protects the organization by making missing setup visible.

Activation is not a broad product-completion claim. It only means this organization has completed the required setup for the selected flow.

## Example

Community Grants DAO cannot activate until it has:

- an admin group;
- a reviewer group;
- an approver group;
- an executor role;
- a grants template;
- required evidence fields;
- a due-date rule for funded work.

Once those are complete, the organization activates and can begin using the grants flow.
