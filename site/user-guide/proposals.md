# Proposals And Decisions

A proposal asks the organization to decide something.

A decision records the outcome.

## A Good Proposal Includes

- title;
- purpose;
- requested action;
- responsible person or group;
- budget or resource request, if any;
- expected evidence;
- due date or expected window;
- review notes;
- risks or open questions.

## Basic Flow

1. Create a proposal.
2. Fill in the required fields from the template.
3. Submit it for review.
4. Reviewers request changes or mark it ready.
5. The organization decides under the configured route.
6. The record shows the outcome.
7. If approved, the record creates or updates the expected follow-up.

## Possible Outcomes

| Outcome | Meaning |
| --- | --- |
| Approved | The proposal passed under the configured process. |
| Rejected | The proposal did not pass. |
| Cancelled | The proposal was intentionally stopped. |
| Expired | The decision window passed without completion. |
| Blocked | The proposal cannot move forward until a problem is resolved. |

## Example

Community Grants DAO reviews a proposal to fund a contributor writing public documentation. Reviewers ask for clearer deliverables. The contributor updates the proposal. Approvers approve it. The decision creates an accountability record with the contributor, due date, and required evidence.
