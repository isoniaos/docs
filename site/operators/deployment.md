# Deployment

Deployment guidance is being normalized with the core repositories.

This page describes deployment categories without publishing stale commands as current instructions.

## Deployment components

A deployment may include:

- contracts deployed to a supported chain;
- Control Plane services for indexing, projections, APIs, and diagnostics;
- database infrastructure;
- App Core static or hosted frontend;
- runtime configuration;
- wallet connector settings;
- optional integration credentials;
- monitoring and backups.

## Environment posture

Future hosted environments should distinguish:

- development;
- stage;
- production later, after stronger operational controls.

Public docs currently do not claim production operations or managed SaaS completion.

This area is being normalized in the current workspace cycle. Use repository-local docs for exact deployment commands until this page is updated with a supported path.
